# 온디바이스 AI (demo-ondevice-ai)

**이 기기·브라우저에서 지금 쓸 수 있는 온디바이스 AI가 무엇인지** 탐지하고, 실제로 동작하는지 검증하는 앱.

- Chrome 내장 AI(Prompt API·Gemini Nano, Summarizer, Translator, Language Detector) + 오리진 트라이얼(Writer·Rewriter·Proofreader) + 기반 기술(WebGPU·WebNN)을 **3단계 진단**: API 존재 → 사용 가능 상태 → 원탭 실동작 검증(지연시간 측정)
- 모델이 없으면 "다운로드 필요"로 표시하고, 다운로드는 **버튼을 눌렀을 때만** 진행률과 함께 실행
- 진단 결과를 텍스트로 복사해 공유 가능
- **모든 진단이 브라우저 안에서만 실행** — 서버로 아무것도 보내지 않음

> 이 앱은 **"회사원도 이런 앱을 만들 수 있다"** 교육용 데모 시리즈의 하나입니다.
> 폰에서 열면 대부분 "API 없음"이 나오는데, 그게 오답이 아니라 **오늘의 플랫폼 지형**입니다 —
> 온디바이스 AI는 지금 데스크톱 Chrome 계열에서 먼저 열리고 있습니다.

## 🙋 이 앱, 나도 만들 수 있을까?

**네, 코딩을 몰라도 됩니다.** 이 앱도 AI(Claude)에게 말로 시켜서 만들었습니다. 핵심 아이디어는 "기능을 만들기 전에, **내 환경에서 뭐가 되는지부터 확인하는 도구**를 만든다"였습니다.

### 따라 만들기 — 이 프롬프트를 그대로 써보세요

```
브라우저의 온디바이스 AI 지원 현황을 진단하는 웹앱을 만들어줘.

- Chrome 내장 AI API들(LanguageModel/Prompt API, Summarizer, Translator,
  LanguageDetector, Writer, Rewriter, Proofreader)과 WebGPU, WebNN을 점검
- 항목마다 3단계로: ① API가 존재하는지 → ② availability()가
  사용가능/다운로드필요/불가 중 무엇인지 → ③ 사용 가능하면 "검증" 버튼으로
  실제 호출을 해보고 결과와 걸린 시간(ms)을 보여줘
- 주의: 모델 다운로드(수 GB)를 유발할 수 있는 create() 호출은
  자동으로 하지 말고, 사용자가 버튼을 눌렀을 때만 진행률과 함께 실행
- 브라우저·OS·메모리 정보를 함께 보여주고, 결과를 텍스트로 복사하는 버튼
- API가 아예 없는 브라우저(아이폰 등)에서도 열리고, "없음"을 정직하게 표시
- 어두운 배경, 모바일 우선
```

**포인트**: AI API는 브라우저·버전·기기마다 지원이 다르고 빠르게 바뀝니다. 그래서 "덮어놓고 기능부터"가 아니라 **탐지 → 검증 → 그다음에 기능**이 올바른 순서고, 이 앱이 그 첫 단계를 맡습니다.

### 더 쉬운 방법 — 이 저장소를 통째로 보여주기

```
src/capabilities.js에 새 API 점검 항목을 추가해줘.
[API 이름]이 최근 크롬에 들어왔다고 하니 같은 3단계 구조로 진단하게.
```

## 📚 알아두면 좋은 개념 (비개발자용 최소 지식)

| 개념 | 한 줄 설명 |
|---|---|
| **온디바이스 AI** | 클라우드 서버가 아니라 **내 기기 안에서** 모델이 도는 AI — 데이터가 밖으로 안 나가고, 호출 비용 0 |
| **Gemini Nano** | Chrome에 내장된 구글의 소형 온디바이스 모델 (Edge는 Phi 계열) |
| **Prompt API** | 웹페이지가 브라우저 내장 모델에게 자유 질문을 보내는 표준 창구 |
| **availability()** | "사용 가능 / 다운로드 필요 / 이 기기에선 불가"를 알려주는 사전 점검 함수 |
| **오리진 트라이얼** | 정식 출시 전 API를 신청 사이트에서만 미리 열어주는 실험 제도 |
| **WebGPU** | 브라우저에서 GPU 연산을 쓰는 표준 — 모델을 직접 내려받아 돌리는 방식(WebLLM류)의 기반 |
